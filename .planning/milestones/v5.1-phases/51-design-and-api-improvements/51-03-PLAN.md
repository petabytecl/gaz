  <action>
    1. **(DSGN-03) Cron context propagation** — In `app_build.go`, function `initializeSubsystems()`:
       The scheduler currently receives `context.Background()`. However, we need to pass the app lifecycle context. Since `initializeSubsystems` doesn't receive a context, add a `runCtx context.Context` field to the `App` struct (set in `Run()` before calling subsystem start methods). Then in `initializeSubsystems`, use `context.Background()` still for creation but ensure the scheduler's `Start()` (called in `Run()`) receives the app context. Check how `cron.NewScheduler` uses the context — if it stores it for job execution, pass a derived context. If the scheduler creates its own context from what's passed at creation time, then the fix is at the `NewScheduler` call site: store a cancel func and pass the derived context.

       Simplest correct fix: In `initializeSubsystems()`, change:
       ```go
       a.scheduler = cron.NewScheduler(a.container, context.Background(), log)
       ```
       to store a cancel context that gets cancelled in Stop:
       ```go
       a.cronCtx, a.cronCancel = context.WithCancel(context.Background())
       a.scheduler = cron.NewScheduler(a.container, a.cronCtx, log)
       ```
       Add `cronCtx context.Context` and `cronCancel context.CancelFunc` fields to App struct (in app.go). Call `a.cronCancel()` in `doStop()` (in app_shutdown.go).

    2. **(DSGN-04) Shutdown error joining** — In `app_run.go`, find the two rollback sites where `_ = a.Stop(shutdownCtx)` discards the error. Change to:
       ```go
       stopErr := a.Stop(shutdownCtx)
       return errors.Join(startupErr, stopErr)
       ```
       There are at least two sites: after service startup errors (~line 911 original) and after worker startup error (~line 922 original). Fix both.

    3. **(DSGN-10) Timer leak fixes** — Two locations:

       a. In `app_shutdown.go`, function `doStop()`, replace:
       ```go
       case <-time.After(a.opts.ShutdownTimeout):
       ```
       with:
       ```go
       timer := time.NewTimer(a.opts.ShutdownTimeout)
       defer timer.Stop()
       ...
       case <-timer.C:
       ```

       b. In `worker/supervisor.go` (line 160), replace:
       ```go
       case <-time.After(delay):
       ```
       with:
       ```go
       timer := time.NewTimer(delay)
       select {
       case <-timer.C:
       case <-s.ctx.Done():
           timer.Stop()
           ...
       }
       ```
       Note: worker/supervisor.go is also modified by plan 01 (DeadLetterInfo). Since these are different functions (runLoop vs invokeDeadLetterHandler), there is no conflict — but the executor should read the latest file state.
