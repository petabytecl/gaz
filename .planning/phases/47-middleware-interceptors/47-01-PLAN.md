Create `server/connect/interceptors.go` with:

1. **Priority constants** — duplicate from gRPC (same values, own package):
   ```go
   const (
       PriorityLogging    = 0
       PriorityRateLimit  = 25
       PriorityAuth       = 50
       PriorityValidation = 100
       PriorityRecovery   = 1000
   )
   ```

2. **ConnectInterceptorBundle interface**:
   ```go
   type ConnectInterceptorBundle interface {
       Name() string
       Priority() int
       Interceptors() []connect.Interceptor
   }
   ```

3. **collectConnectInterceptors()** — mirrors `server/grpc/interceptors.go:76-106`. Takes `*di.Container` and `*slog.Logger`. Uses `di.ResolveAll[ConnectInterceptorBundle]`, sorts by Priority(), flattens `Interceptors()` slices, logs each registered bundle. Returns `[]connect.Interceptor`.

4. **ConnectAuthFunc type**:
   ```go
   type ConnectAuthFunc func(ctx context.Context, req connect.AnyRequest) (context.Context, error)
   ```

5. **ConnectLimiter interface**:
   ```go
   type ConnectLimiter interface {
       Limit(ctx context.Context, req connect.AnyRequest) error
   }
   ```

6. **AlwaysPassLimiter** — implements ConnectLimiter, always returns nil.

7. **LoggingBundle** — hand-written `connect.Interceptor` that logs procedure name, duration, error status via slog. Implements `WrapUnary` (logs after call), `WrapStreamingClient` (pass-through — server-side only), `WrapStreamingHandler` (logs after stream completes). Priority 0.

8. **RecoveryBundle** — hand-written `connect.Interceptor` with defer/recover in both `WrapUnary` and `WrapStreamingHandler`. Logs panic + stack trace via slog. Returns `connect.NewError(connect.CodeInternal, ...)` — dev mode includes panic details, production returns generic message. `WrapStreamingClient` is pass-through. Priority 1000. Constructor takes `*slog.Logger` and `bool` (devMode).

9. **AuthBundle** — `connect.Interceptor` that calls `ConnectAuthFunc` with `ctx` and `req` in `WrapUnary`. On error, returns the error directly (caller should return connect error codes). On success, uses returned context for downstream. `WrapStreamingHandler` extracts request info from `conn.RequestHeader()` — construct a minimal `connect.AnyRequest` adapter or call auth func with headers directly. Constructor takes `ConnectAuthFunc`. Priority 50.

10. **RateLimitBundle** — `connect.Interceptor` that calls `ConnectLimiter.Limit()` in `WrapUnary` and `WrapStreamingHandler`. If nil limiter, uses AlwaysPassLimiter. Priority 25.

11. **ValidationBundle** — wraps `validate.NewInterceptor()` from `connectrpc.com/validate`. Constructor calls `validate.NewInterceptor()` (no error return). Interceptors() returns `[]connect.Interceptor{validateInterceptor}`. Priority 100.

**Import organization:** stdlib → external (`connectrpc.com/connect`, `connectrpc.com/validate`) → local (`github.com/petabytecl/gaz/di`).

**Key differences from gRPC version:**
- Returns `[]connect.Interceptor` (slice) not `(unary, stream)` pair — Connect uses single interceptor type
- AuthFunc takes `connect.AnyRequest` not gRPC metadata context
- Limiter takes `connect.AnyRequest` for per-procedure/per-user decisions
- Recovery uses `connect.NewError(connect.CodeInternal, ...)` not `status.Error(codes.Internal, ...)`
- Logging is hand-written (no go-grpc-middleware logging adapter for Connect)

Create `server/connect/interceptors_test.go` following the test patterns in `server/grpc/interceptors_test.go`. Use testify suites. Test all bundles for interface compliance, priority values, interceptor return. Test collectConnectInterceptors for ordering, empty container, nil handling. Test recovery for panic behavior in dev and prod mode. Test auth opt-in pattern. Test AlwaysPassLimiter.
