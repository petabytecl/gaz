    **di/bench_test.go:**
    ```
    package di_test
    ```
    1. `BenchmarkResolve_Singleton` — register a singleton provider, build container, benchmark `di.Resolve[T](c)` in b.N loop. This measures the cached singleton path (map lookup + type assertion). Use a simple struct as T.
    2. `BenchmarkResolve_Transient` — register a transient provider, build container, benchmark resolve. This measures provider invocation on every call.
    3. `BenchmarkResolve_Named` — register a named singleton, benchmark `di.ResolveNamed[T](c, "name")`. Tests the named lookup path.
    4. `BenchmarkResolve_Parallel` — use `b.RunParallel` to benchmark concurrent singleton resolution. This validates the sync.RWMutex performance under contention.
    5. `BenchmarkResolveAll` — register 10 implementations of an interface, benchmark `di.ResolveAll[T](c)`. Tests the reflection-based discovery path.

    **eventbus/bench_test.go:**
    ```
    package eventbus_test
    ```
    1. `BenchmarkPublish_SingleSubscriber` — create bus, subscribe one handler (that does nothing), benchmark `Publish`. Measures channel send + dispatch overhead.
    2. `BenchmarkPublish_TenSubscribers` — same but with 10 subscribers. Measures fan-out cost.
    3. `BenchmarkPublish_Parallel` — use `b.RunParallel` for concurrent publish. Measures lock contention under load.
    Note: Since Publish is async, the handler goroutines may accumulate. Use `b.StopTimer()` and drain/close between iterations if needed, or accept that the benchmark measures the publish call latency (not handler completion).

    **backoff/bench_test.go:**
    ```
    package backoff_test
    ```
    1. `BenchmarkExponentialBackOff_NextBackOff` — create ExponentialBackOff with defaults, benchmark `NextBackOff()`. Reset between iterations with `b.StopTimer()`/`b.StartTimer()` every N calls to avoid hitting Stop sentinel.
    2. `BenchmarkConstantBackOff_NextBackOff` — baseline comparison, should be near-zero allocation.
    3. `BenchmarkExponentialBackOff_Reset` — benchmark the Reset() call.
