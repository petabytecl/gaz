       ```go
       type eventEnvelope struct {
       ```

       ```go
       func (s *asyncSubscription) run(logger *slog.Logger) {

           }
       }
       ```

  </behavior>
  <action>
    (DSGN-09) Current `OnStart` launches `ListenAndServe` in a goroutine and returns nil immediately. If the port is already in use, the error is only logged.

    Fix: Use `net.Listen` to bind the port synchronously in `OnStart`, then pass the listener to `Serve`:

    1. Add a `listener net.Listener` field to the `Server` struct.

    2. In `OnStart`:
       ```go
       ln, err := net.Listen("tcp", s.server.Addr)
       if err != nil {
           return fmt.Errorf("http server listen: %w", err)
       }
       s.listener = ln
       s.started.Store(true)
       s.logger.InfoContext(ctx, "HTTP server starting", "addr", ln.Addr().String())
       go func() {
           if err := s.server.Serve(ln); err != nil && !errors.Is(err, http.ErrServerClosed) {
               s.logger.Error("HTTP server error", "error", err)
           }
       }()
       return nil
       ```

    3. In `OnStop`, no changes needed — `s.server.Shutdown(ctx)` still works with `Serve`.

    4. If there is a `Port()` or address accessor, update to use `s.listener.Addr()` for the actual bound address (important when port=0).

    This is the standard Go pattern: bind synchronously, serve asynchronously.
  </action>
