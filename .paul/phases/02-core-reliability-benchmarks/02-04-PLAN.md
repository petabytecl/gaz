<objective>
## Goal
Add atomic fast-path to lazySingleton and eagerSingleton GetInstance, eliminating mutex acquisition on the hot path for already-built singletons.

## Purpose
Every singleton resolution currently acquires an exclusive mutex even when returning a cached instance. The atomic fast-path (double-checked locking with atomic.Bool) is a well-known optimization that eliminates this contention.

## Output
Modified di/service.go with atomic.Bool built flag. Updated Start/Stop methods to use atomic check.
</objective>
